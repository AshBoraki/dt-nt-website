2. DTNT app data

DTNT is designed primarily as a local Windows tool. Network scan results, Wi-Fi profile data, saved exports, activation records, purchase-email plus activation-code state, fallback activation files, and local diagnostic logs are generally stored on your device unless you choose to export, email, upload, or otherwise share them.

When you use the Profiles workspace, DTNT can read saved Windows Wi-Fi profile names, security labels, and saved keys from Windows so it can show the local profile state on that machine. Saved keys are shown or copied locally when you choose to reveal them, and they are not included in the DTNT profile export format by default.

When you run IP scanner and related discovery workflows, DTNT can send ping, DNS, TCP-connect, ARP-assisted, and short HTTP(S) requests across the local range you select in order to identify responsive devices and likely web interfaces. DTNT may temporarily read headers and a small body snippet to classify a local login page, but that body sample is not kept as a saved export field by default.

Local diagnostic logs can contain timestamps, component names, status messages, and exception details produced by the app on your device.

Some DTNT features necessarily communicate with third-party endpoints that you trigger, such as payment providers, download hosts, update metadata, speed-test services, DNS or web targets, or links you open from inside the app.
